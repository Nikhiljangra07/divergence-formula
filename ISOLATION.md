# ISOLATION.md — Divergent-model project ↔ LoRa project boundary

*Audit date: 2026-07-05. Full scan; rerun this audit if either project starts consuming the other's data.*

Two independent projects, two models, two data estates:

- **Divergent-model** (this repo, `~/Desktop/divergence-formula/`): granite-4.0-h-small "refraction"
  specialist. Training data = `round2_kit/data_*` (100% synthetic, generated by DeepSeek V4 Pro from
  short SOURCE BRIEFS written for this project). Artifacts: `~/Desktop/divergent-model-backups/`.
- **LoRa** (product: `LoRa-EmotionalEngine-v1` + `presence-whispers`; V2 training corpus:
  `~/Desktop/lora-corpus-source/`): the analytical-reasoning product and its multi-civilization
  corpus program.

## Audit results (2026-07-05)

1. **Code references**: divergence-formula contains ZERO code paths into any LoRa repo. One comment
   in `corpus_run/config.py:60` *mentions* `lora-corpus-source/english/diplomacy-and-war/` as the
   inspiration for the Thucydides source brief — documentation only, never read by code.
2. **Data ingestion**: all corpus generators (gen_v2..v5, gen_essence_*) were scanned — none opens any
   file outside this repo. Problems/threads are generated from ~5-sentence SOURCE BRIEFS (original
   prose written for this project), never from raw novel/history text.
3. **Text contamination**: every 8-word shingle of the shipped BLEND2 training set (1,034,825 shingles
   from 6,745 rows) was checked against all 1,412 text files in lora-corpus-source (english, chinese-en,
   sanskrit-en, greek, latin, *-en dirs): **0 collisions**. No sentence of LoRa's corpus texts appears
   in divergent-model training data, and vice versa.
4. **Reverse direction**: lora-corpus-source and the LoRa backend contain no divergence-formula
   artifacts or references (only dictionary-word hits in an emotion lexicon).
5. **Artifact separation**: adapters/evals live in `divergent-model-backups/` only; LoRa corpus work
   lives in `lora-corpus-source/` only.

## Documented cross-touchpoints (allowed, but named — per Nikhil's rule)

- **Idea-level overlap, no data flow**: Jin Ping Mei and Reverend Insanity appear as reasoning
  traditions in BOTH projects (LoRa V2 civilization corpus; divergence essence round). Each project
  wrote its own material: LoRa V2 synthesizes from the actual translated chapters; divergence uses
  original 5-sentence essence briefs (`corpus_run/config_essence.py`). Proven disjoint by check #3.
- **JPM chapter repair (2026-07-04)**: 10 broken chapters of `lora-corpus-source/chinese-en/
  plum-in-the-golden-vase/` were repaired (script: `lora-corpus-source/scripts/repair_jpm_en.py`).
  That work belongs to and benefits the LORA corpus estate; it was merely performed during a
  divergent-model session. No repaired text entered any divergence training set (check #3 postdates
  the repair).
- **Shared credentials**: both projects (and Constellax) load API keys from
  `~/Desktop/reasoningEngine/.env`. Shared billing accounts, not shared data.

**Verdict: SAFE — no entanglement.** Re-run checks 1–4 before shipping either model if cross-use
is ever introduced deliberately.
